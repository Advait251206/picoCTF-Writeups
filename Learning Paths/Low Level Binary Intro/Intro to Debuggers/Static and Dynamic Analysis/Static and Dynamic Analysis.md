## Problem Description
What we did in Bit-O-Asm was static analysis. Static analysis is quite simply reading code. We’re about to introduce a dynamic analysis technique. Dynamic analysis runs code to find out what it does. Sometimes, rather than pouring over every line in a program, we can gain insight quickly by running it under controlled conditions.
Mark Complete

## Solution
This is an informational checkpoint challenge introducing the difference between static and dynamic analysis.

- **Static Analysis:** Examining code (source code, bytecode, or assembly) without executing it. This is what you do when you read through the instructions of a binary to figure out what it's supposed to do (like in the Bit-O-Asm challenges).
- **Dynamic Analysis:** Executing the code in a controlled environment (like a sandbox or a debugger such as GDB) and observing its behavior, memory, and registers in real-time. This is often faster for determining complex logic or seeing the actual values calculated during execution.
