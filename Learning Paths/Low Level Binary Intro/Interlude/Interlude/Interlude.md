## Problem Description
We’re going to take a break from low level code and consider some Python programs again. We’ll try to draw some parallels between exploiting binary programs and exploiting Python programs. We won’t include walkthroughs before the challenges, but we’ll include solution modules after the challenges if you need to sneak a peek. After this Python interlude, we’ll get back to some binary exploitation challenges that have been featured in picoCTF competitions.

## Solution
This is an informational checkpoint marking our transition into the **Interlude** problem set!

We're shifting gears temporarily from raw assembly (`x86-64`, `GDB`, etc.) back to Python. The goal here will be to conceptually bridge the gap between higher-level logic bugs in interpreted languages (like Python) and lower-level memory corruption bugs in compiled binaries (like C/C++). 

By understanding exploitation at a high level first, it often becomes much easier to reason about the exact same vulnerabilities occurring at the memory level.
